Nó là gìWhat it is

Phơi MỌI callback Rithmic R|API+ ra WebSocket theo kiểu 1-1. Các stream curated (@trade, @depth, @mbo, @kline, @bookticker, @mboraw…) là bản đã biến đổi nằm trên lớp này — dùng chúng để vẽ chart. Dùng raw bus khi cần event Rithmic thô, đủ field. Exposes every Rithmic R|API+ callback over the socket 1-to-1. The curated streams (@trade, @depth, @mbo, @kline, @bookticker, @mboraw…) are transforms built on top — use those for charting. Use the raw bus when you want the untouched Rithmic event with all its fields.

Dạng stream (push → stream)Stream form (push → stream)

Subscribe <exch>.<sym>@<TenCallback> (không phân biệt hoa thường). Tên là đúng tên callback R|API+: Subscribe <exch>.<sym>@<CallbackName> (case-insensitive). The name is exactly the R|API+ callback:

SUBSCRIBE  cme.esu6@onsettlementprice
SUBSCRIBE  cme.esu6@onmarketmode
SUBSCRIBE  cme.esu6@onopeninterest
SUBSCRIBE  *.*@onpnlupdate          // toan cuc / global
SUBSCRIBE  *.*@onfillreport

Mỗi message = struct Info phản chiếu sang JSON — e = tên callback, rồi tất cả field public: Each message = the Info struct reflected to JSON — e = callback name, then all public fields:

Bật feed thế nàoWhat turns the feed on

  • Stream theo symbol tự gọi Subscribe(All) → có ngay mọi callback L1 / giá / settlement / limit / market-mode / OI / volume / indicator / reference. A symbol raw stream calls Subscribe(All) → you immediately get every L1 / price / settlement / limit / market-mode / OI / volume / indicator / reference callback.
  • Callback book / depth-by-order / bar (OnLimitOrderBook, OnBidQuote, OnAskQuote, OnDbo, OnBar…) cần feed riêng — sub kèm stream curated bật nó: @depth (book), @mbo/@mboraw (DBO), @kline (bar). Raw bus sẽ chuyển tiếp event của chúng. Book / depth-by-order / bar callbacks need their own feed — subscribe the curated stream that activates it alongside: @depth (book), @mbo/@mboraw (DBO), @kline (bars). The raw bus then passes their events through.
  • Callback query-response (OnOrderReplay, OnExecutionReplay, OnOptionList, OnRefData…) KHÔNG tự đẩy — phải gọi 1 method để kích. Raw bus tự gọi giùm khi bạn sub (xem bảng trigOf). Lưu ý: tên method ≠ tên callback — đây là cặp request/response của R|API+: bạn gọi REngine.replayX(), Rithmic bắn lại callback XReplay. Phần lớn khớp (replayOpenOrders→OnOpenOrderReplay) NHƯNG có chỗ lệch — replayAllOrders() → OnOrderReplay (method có "All", callback không) — nên không suy ra được bằng quy tắc chuỗi, phải tra trigOf. Tiền tố On + viết-thường là quy ước stream của gateway này; R|API+ gốc gọi callback là OrderReplay (không "On"). Query-response callbacks (OnOrderReplay, OnExecutionReplay, OnOptionList, OnRefData…) do NOT self-push — a method must trigger them. The raw bus calls it for you on subscribe (see the trigOf table). Note: method name ≠ callback name — it's R|API+'s request/response pair: you call REngine.replayX(), Rithmic fires back the XReplay callback. Most line up (replayOpenOrders→OnOpenOrderReplay) BUT some don't — replayAllOrders() → OnOrderReplay (method has "All", callback doesn't) — so it can't be derived by a string rule, you must look it up in trigOf. The On-prefix + lowercasing is this gateway's stream convention; raw R|API+ names the callback OrderReplay (no "On").
  • Stream toàn cục tự chảy sau login — không cần bật.Global streams flow automatically after login.
Chi phí: passthrough chỉ chạy khi có client giữ event đó (ref-count); không ai sub thì 0 việc. Callback tần suất cao (OnQuote, OnBidQuote, OnAskQuote) nặng — chỉ sub cái bạn xử lý. Cost: the passthrough runs only while a client holds that event (ref-counted); nobody subscribed = zero work. High-frequency callbacks (OnQuote, OnBidQuote, OnAskQuote) are heavy — subscribe only what you consume.
