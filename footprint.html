Footprint là gìWhat is a footprint

Một footprint (còn gọi order-flow / cluster / bid-ask chart) tách mỗi cây nến theo mức giá, và ở từng mức cho thấy khối lượng khớp ở phía mua (ask) so với phía bán (bid). Nó biến 1 cây nến thành một cột các hàng giá như sau: A footprint (a.k.a. order-flow / cluster / bid-ask chart) breaks each candle down by price level and shows, at every level, how much volume traded on the buy (ask) side vs the sell (bid) side. It turns a single candle into a column of price rows like:

85501.0031
425500.7595
605500.5018
125500.254

Trái = khối lượng bán · giá ở giữa · phải = khối lượng mua. Hàng vàng là POC (giá khớp nhiều nhất). Từ đây đọc được sự hấp thụ, mất cân bằng và delta. Left = sell volume · price in the middle · right = buy volume. The yellow row is the POC (most-traded price). From this you read absorption, imbalances and delta.

Cần gọi socket nàoWhich sockets to call

Footprint dựng hoàn toàn từ các lệnh khớp riêng lẻ có kèm bên chủ động. Bạn chỉ cần đúng hai nguồn từ WebSocket API — một cho lịch sử, một cho realtime: A footprint is built entirely from individual trades that carry an aggressor side. You need exactly two data sources from the WebSocket API — one for history, one for live:

Thuật toánAlgorithm

Footprint là một bảng đếm lồng nhau: bucket[barStart][price] = { buy, sell }. Footprint is a nested tally: bucket[barStart][price] = { buy, sell }.

1Chọn intervalPick the interval và tính độ dài theo ms, vd 1m → 60000.and compute its length in ms, e.g. 1m → 60000.

2Tìm nến của mỗi tradeFind each trade's bar bằng cách làm tròn xuống timestamp:by flooring its timestamp:

barStart = Math.floor(timeMs / intervalMs) * intervalMs;

3Cộng trade vào mức giáAdd the trade to its price level, tách theo side:, split by side:

cell = bucket[barStart][price]            // {buy:0, sell:0}
if (side === "B") cell.buy  += qty
else if (side === "S") cell.sell += qty
else { /* unknown -> tick rule, xem Pitfalls */ }

4Nạp lịch sử, rồi cộng live.Seed history, then keep adding live. Replay aggTrades một lần để điền nến quá khứ, sau đó áp dụng mỗi @trade y hệt — nến hiện tại lớn dần theo từng tick.Replay aggTrades once to fill past bars, then apply every @trade the same way — the current bar grows tick by tick.

5VẽRender mỗi nến thành một cột: mỗi mức giá đã khớp là một hàng, hiển thị bán | mua, tô màu theo khối lượng, đánh dấu POC/delta.each bar as a column: one row per price that traded, showing sell-vol | buy-vol, colored by volume, with POC/delta highlighted.

Ví dụ đầy đủ — PythonFull example — Python

Dựng footprint live: replay 1 giờ lịch sử, rồi stream trade.Builds a live footprint: replays one hour of history, then streams trades.

import asyncio, json, time, math, collections, websockets

URL="ws://127.0.0.1:9001/"; EXCH="CME"; SYM="ESU6"; IV_MS=60_000   # 1-minute bars
# bucket[barStart][price] = [buy, sell]
book = collections.defaultdict(lambda: collections.defaultdict(lambda: [0,0]))

def add(t_ms, price, qty, side, last=[None]):
    if side not in ("B","S"):                     # tick-rule fallback
        side = "B" if last[0] is None or price >= last[0] else "S"
    last[0] = price
    bar = (t_ms // IV_MS) * IV_MS
    cell = book[bar][price]
    cell[0 if side=="B" else 1] += qty

def show(bar):
    levels = book[bar]
    buy  = sum(c[0] for c in levels.values())
    sell = sum(c[1] for c in levels.values())
    poc  = max(levels, key=lambda p: sum(levels[p]))
    print(f"bar {time.strftime('%H:%M', time.localtime(bar/1000))}  "
          f"D={buy-sell:+d}  POC={poc}  vol={buy+sell}")

async def main():
    async with websockets.connect(URL, max_size=None) as ws:
        now = int(time.time()*1000)
        # 1) history -> seed footprint
        await ws.send(json.dumps({"method":"aggTrades","id":1,"params":{
            "exchange":EXCH,"symbol":SYM,"startTime":now-3600_000,"endTime":now}}))
        # 2) live trades
        await ws.send(json.dumps({"method":"SUBSCRIBE","id":2,
            "params":[f"{EXCH}.{SYM}@trade"]}))
        async for raw in ws:
            m = json.loads(raw)
            if m.get("id")==1 and m.get("result"):                 # aggTrades reply
                for t,p,q,s in m["result"]: add(t,p,q,s)
                print("seeded", len(m["result"]), "trades")
            elif m.get("stream","").endswith("@trade"):            # live trade
                d=m["data"]; add(d["T"], d["p"], d["q"], d["side"])
                show((d["T"]//IV_MS)*IV_MS)

asyncio.run(main())

Ví dụ đầy đủ — JavaScriptFull example — JavaScript

const URL="ws://127.0.0.1:9001/", EXCH="CME", SYM="ESU6", IV_MS=60000;
const book = new Map();                      // barStart -> Map(price -> {buy,sell})
let last = null;
function cell(bar, price){
  if(!book.has(bar)) book.set(bar, new Map());
  const m = book.get(bar);
  if(!m.has(price)) m.set(price, {buy:0, sell:0});
  return m.get(price);
}
function add(t, price, qty, side){
  if(side!=="B" && side!=="S") side = (last===null || price>=last) ? "B" : "S";  // tick rule
  last = price;
  const bar = Math.floor(t/IV_MS)*IV_MS;
  const c = cell(bar, price);
  if(side==="B") c.buy += qty; else c.sell += qty;
}
function metrics(bar){
  const m = book.get(bar); if(!m) return;
  let buy=0, sell=0, poc=null, pv=-1;
  for(const [p,c] of m){ buy+=c.buy; sell+=c.sell; const t=c.buy+c.sell; if(t>pv){pv=t;poc=p;} }
  console.log(`D=${buy-sell}  POC=${poc}  vol=${buy+sell}`);
}

const ws = new WebSocket(URL); let id=1;
ws.onopen = () => {
  const now = Date.now();
  ws.send(JSON.stringify({method:"aggTrades", id:1, params:{
    exchange:EXCH, symbol:SYM, startTime:now-3600000, endTime:now}}));
  ws.send(JSON.stringify({method:"SUBSCRIBE", id:2, params:[`${EXCH}.${SYM}@trade`]}));
};
ws.onmessage = (e) => {
  const m = JSON.parse(e.data);
  if(m.id===1 && m.result){ m.result.forEach(([t,p,q,s])=>add(t,p,q,s)); }
  else if((m.stream||"").endsWith("@trade")){
    const d=m.data; add(d.T, d.p, d.q, d.side); metrics(Math.floor(d.T/IV_MS)*IV_MS);
  }
};

Bẫy thường gặpPitfalls

  • Phân loại side. Ưu tiên side từ server ("B"/"S"). Khi nó là "", dùng tick rule: giá ≥ trade trước → mua, ngược lại → bán (như trong ví dụ). Chính xác hơn thì so với bid/ask tốt nhất từ @bookTicker. Side classification. Prefer the server's side ("B"/"S"). When it is "", fall back to the tick rule: price ≥ last trade → buy, else sell (used in the examples). For better accuracy, compare against best bid/ask from @bookTicker.
  • Căn nến. floor(time/intervalMs)*intervalMs căn theo epoch, khớp ranh giới phút/giờ. Nến tick/volume/range không căn theo thời gian — footprint theo thời gian chỉ hợp lý với interval thời gian (s/m/h/d). Bar alignment. floor(time/intervalMs)*intervalMs aligns to the epoch, which matches minute/hour boundaries. Tick/volume/range bars are not time-aligned — footprint by time only makes sense for time-based intervals (s/m/h/d).
  • Lịch sử + live chồng lấn. aggTrades kết thúc ở "now" và @trade live tiếp tục từ đó; ranh giới liền mạch. Đừng đếm trùng bằng cách replay hai lần cùng một khoảng. History + live overlap. aggTrades ends at "now" and live @trade continues from there; the boundary is seamless. Don't double-count by replaying an overlapping window twice.
  • Giá float làm key. Giá khớp là bội số tick chính xác (vd 5500.25); dùng số làm key map là ổn. Hiển thị số lẻ theo tick của mã, đừng cố định. Float prices as keys. Trade prices are exact tick multiples (e.g. 5500.25); using the number as a map key is fine. Format display decimals from the instrument's tick, not a fixed value.
  • Tick replay chỉ có cho phiên LIVE đang chạy. aggTrades dựa trên tick-replay nội phiên của Rithmic, chỉ phục vụ trade của phiên đang mở. Khi phiên đã đóng (qua đêm, cuối tuần, nghỉ lễ) thì tick từng lệnh không còn replay được — aggTrades trả rỗng kể cả với cửa sổ nằm gọn trong phiên cũ đó. Đây là giới hạn của data-plant Rithmic, không phải lỗi, và nới cửa sổ thời gian không lấy lại được. Nến OHLC (klines) thì vẫn có lịch sử — chỉ phần tách mua/bán theo từng lệnh là không. Hệ quả: footprint lịch sử chỉ có khi thị trường đang mở; khi aggTrades rỗng, báo cho người dùng là chợ đóng và dựng footprint realtime từ @trade khi giao dịch trở lại — đừng coi replay rỗng là lỗi. Tick replay only covers the live/current session. aggTrades is backed by Rithmic's intraday tick replay, which serves trades only for the session that is currently open. Once a session closes (overnight, weekends, holidays) its per-trade ticks are no longer replayable — aggTrades returns empty even for a window squarely inside that past session. This is a Rithmic data-plant limitation, not a bug, and widening the window does not recover them. OHLC bars (klines) are available historically — only the per-trade buy/sell split is not. Consequence: historical footprint is only available while the market is open; when aggTrades returns empty, tell the user the market is closed and build footprint live from @trade once trading resumes — don't treat the empty replay as an error.
