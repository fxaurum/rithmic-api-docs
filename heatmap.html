Heatmap là gìWhat it is

Heatmap thanh khoản theo thời gian × giá, kiểu Bookmap: mỗi cột pixel là một thời điểm, mỗi hàng là một giá; độ sáng ô = size lệnh chờ tại (thời gian, giá). Vẽ đè thêm: đường best bid/ask, bóng trade, và histogram sổ lệnh hiện tại ở mép phải. Là một primitive LWC nằm dưới nến (bật "Hiện nến" thì footprint/nến đè lên). A liquidity heatmap over time × price, modelled on Bookmap: each pixel column is a moment, each row a price; cell brightness = resting order size at that (time, price). Overlaid: a best bid/ask line, trade bubbles, and a current order-book depth histogram on the right edge. An LWC primitive drawn under the candles (turn on "show candles" to overlay footprint/candles).

Dải màu (ít → nhiều thanh khoản): tối → xanh dương → cyan → trắng → vàng → cam → đỏ. Gamma nâng tương phản. Colormap (low → high liquidity): dark → blue → cyan → white → yellow → orange → red. Gamma boosts contrast.

Cần gọi socket nàoWhich sockets to call

Heatmap ghép 3 nguồn LIVE (không replay lịch sử — dựng tiến từ lúc bật): The heatmap combines 3 live sources (no history replay — it builds forward from when you open it):

Đường bid/askBid/ask line@bookTicker Vẽ từ mỗi thay đổi quote (không phải mẫu 300 ms) -> đường sát, bóng trade khớp đúng.From every quote change (not the 300 ms sample) so the line is tight and bubbles sit on it.
Bóng tradeTrade bubbles@trade Một chấm mờ mỗi lệnh; bán kính ∝ √qty; xanh = mua / đỏ = bán.A translucent dot per print; radius ∝ √qty; blue = buy / red = sell.

Tối thiểu = @depth (ô + histogram) + @bookTicker (đường). @trade chỉ thêm bóng. Depth cần quyền L2 từ Rithmic. Minimum = @depth (cells + histogram) + @bookTicker (line). @trade only adds bubbles. Depth needs L2 entitlement.

Các lớp vẽ (dưới → trên)Layers (bottom → top)

  1. Ô nhiệt — mỗi cột 300 ms, mỗi mức sổ là 1 ô màu. Sáng = log(size+1)/log(max) mũ gamma. Cột lấp đầy từ cột trước (không hở sọc đen). Luôn trải full bề ngang pane. Heat cells — per 300 ms column, each book level is a colored rectangle. Brightness = log(size+1)/log(max) ^ gamma. Columns gap-fill (no black stripes). Always stretched to full pane width.
  2. Đường bid/ask — step line từ @bookTicker. Trái vạch vàng = nét liền; phải (vùng sổ lệnh) = nét đứt. Chưa có tick mới thì giữ giá cuối kéo ngang tới mép phải. Bid/ask line — step line from @bookTicker. Left of the yellow line = solid; right (depth zone) = dashed. Holds the last value to the right edge when no new tick.
  3. Sổ lệnh hiện tại (Bookmap mép phải) — thanh ngang của sổ mới nhất: đỏ = ask (trên giá), xanh = bid (dưới). Dài ∝ size; cột to nhất chạm "vạch vàng". Current order-book depth (Bookmap right edge) — horizontal bars of the latest book: red = ask (above), green = bid (below). Length ∝ size; the largest bar reaches the "yellow line".
  4. Bóng trade — vòng tròn mờ tại (thời gian, giá) của lệnh.Trade bubbles — translucent circles at (trade time, price).

Bên trongInternals

  • Sổ lệnh là cùng DomBook với DOM (dựng từ OnBidQuote/OnAskQuote + snapshot, phát @depth<N> ~30 Hz). Client lấy mẫu mỗi HEAT_MS = 300 ms -> heatCols. The book is the same DomBook as the DOM (from OnBidQuote/OnAskQuote + snapshot, emitted @depth<N> ~30 Hz). The client samples every HEAT_MS = 300 ms -> heatCols.
  • Đường bid/ask dùng mảng riêng heatQuotes theo từng @bookTicker (mịn hơn ô 300 ms) -> bóng khớp đường. X là stretch full-width (T−viewStart)/viewMs × paneWidth (LWC không có trục X dưới-cây-nến nên vẽ tay). The bid/ask line uses a separate heatQuotes fed by each @bookTicker (finer than the 300 ms cells) -> bubbles align. X is a full-width stretch (T−viewStart)/viewMs × paneWidth (LWC has no sub-bar X axis, so it's hand-drawn).
  • Xem STORAGE: cách @depth<N> cắt theo từng subscription để DOM + heatmap dùng chung 1 stream ở độ sâu chọn. See STORAGE: how @depth<N> is trimmed per subscription so DOM + heatmap share one stream at a chosen depth.
