Indicator & Order-Flow RecipesIndicator & Order-Flow Recipes

Một cookbook: với mỗi chỉ báo futures phổ biến — dùng stream/request nào của API và thuật toán để tính. Mỗi recipe tự chứa, không phụ thuộc ngôn ngữ — dựng từ WebSocket API bằng JS, Python, C#, gì cũng được. Mục tiêu: cho người dùng một tham chiếu để xây chỉ báo, rồi từ đó xây chiến lược/bot. A cookbook: for each popular futures indicator, which API streams/requests to use and the algorithm to compute it. Each recipe is self-contained and language-agnostic — build it from the WebSocket API in JS, Python, C#, anything. The goal is to give end users a reference to build indicators and, from there, trading strategies/bots.

1 · Big trades / phát hiện blockBig trades / block detection ✓

Đánh dấu các print lớn bất thường — size tổ chức, sweep, iceberg lộ diện. (Mô phỏng theo ATAS Big trades.) Flag unusually large prints — institutional size, sweeps, icebergs surfacing. (Modelled on ATAS Big trades.)

Dữ liệu:Data: @trade (live) + aggTrades (lịch sử để back-fill markerhistory to back-fill markers).

Hai chế độTwo modes

  • Cumulative (mặc định): gộp các print thuộc cùng một matching event thành một trade logic, rồi lọc theo tổng size. Một lệnh chủ động quét nhiều mức giá hiện ra MỘT marker, không phải cụm print nhỏ. Đây là cách ATAS CumulativeTrades hoạt động. (default): merge prints that belong to the same matching event into one logical trade, then threshold the summed size. A single aggressive order that swept several price levels shows as ONE marker, not a cluster of small prints. This is how ATAS CumulativeTrades works.
  • Separate: lọc ngưỡng từng print riêng lẻ.: threshold each individual print.

Gộp nhóm — dùng mốc matching event (tu).Grouping — use the matching-event timestamp (tu). Một matching event (một cú quét) đóng dấu mọi print cùng một transact time của sàn; gateway đẩy ra ở tu (micro giây). Gộp dãy print liên tiếp cùng tu = một cumulative trade — đúng khoá mà CumulativeTrade của ATAS dùng (.Time). Rithmic cũng cấp o (AggressorExchOrdId) nhưng đừng lấy làm khoá chính: một lệnh có thể khớp rải qua nhiều event (lệnh nằm chờ/iceberg ăn dần), gộp theo o sẽ gộp dồn thành cụm to mà ATAS tách riêng (đo được lệch cao hơn ATAS). Chỉ dùng o dự phòng khi thiếu tu; còn khi chỉ có ms (replay store cũ) thì cửa sổ thời gian (cùng side trong sweepMs, vd 50 ms). Thứ tự: tu → o → cửa sổ thời gian. A matching event (one aggressing sweep) stamps every print with the same exchange transact time; the gateway forwards it as tu (microseconds). Group the run of consecutive prints with an identical tu into one cumulative trade — exactly what ATAS's CumulativeTrade keys on (its .Time). Rithmic also supplies o (AggressorExchOrdId) but don't make it the primary key: one order can fill across several match events (a resting/iceberg order eaten over time), and grouping by o merges those into one oversized trade that ATAS shows separately (measured to drift higher than ATAS). Use o only as a fallback when tu is absent; a time window (same side within sweepMs, e.g. 50 ms) when only millisecond time survives (old store replay). Order: tu → o → time window.

Thuật toán (cumulative)Algorithm (cumulative)

  1. Chọn size min (và max tuỳ chọn, 0 = không trần).Pick min size (and optional max, 0 = no cap).
  2. Duyệt trade theo thời gian với nhóm hiện hành {side, vol, o, tu, tLast, tArr}.Walk trades in time order with a current group {side, vol, o, tu, tLast, tArr}.
  3. Cùng nhóm nếu print có tu bằng tu của nhóm (cùng matching event); hoặc cùng o (cùng lệnh aggressor, dự phòng); nếu không (chỉ có ms) thì cùng side và trong sweepMs kể từ tLast. Khớp → cộng; ngược lại chốt (phát nếu min ≤ vol và max ≤ 0 || vol < max) rồi mở nhóm mới. Same group if the print's tu equals the group's tu (same matching event); else equal o (same aggressor order, fallback); else (ms only) same side and within sweepMs of tLast. Match → add; otherwise flush (emit if min ≤ vol and max ≤ 0 || vol < max) and start a new group.
  4. Chốt nhóm cuối ở cuối danh sách (lịch sử) hoặc sau khoảng hở > sweepMs (live: dùng timer). Đo khoảng hở live theo mốc tới máy, không phải timestamp sàn, để độ trễ feed không chốt nhóm non rồi cắt đuôi sweep.Flush the final group at end of list (history) or after a gap > sweepMs (live: a timer). Measure the live gap from arrival time, not the exchange timestamp, so feed latency can't close a group early and clip a sweep's tail.
  5. Vẽ marker (▲ dưới nến cho mua, ▼ trên nến cho bán) hoặc bong bóng theo khối lượng; nhãn = size nhóm.Render markers (▲ below bar for buy, ▼ above for sell) or bubbles sized by volume; label = group size.
function buildBigTrades(trades, {min, max=0, sweepMs=50, cumulative=true}){  // trades: [{T,q,side,tu,o}] sorted by T
  const ok = v => v>=min && (max<=0 || v<max), out=[];
  if(!cumulative){ for(const t of trades) if(ok(t.q)) out.push({T:t.T, q:t.q, side:t.side||""}); return out; }
  let g=null; const flush=()=>{ if(g && ok(g.q)) out.push({T:g.T, q:g.q, side:g.side}); g=null; };
  const same=(t,s)=> g && ((t.tu&&g.tu) ? t.tu===g.tu              // cùng matching event (µs) — khoá ATAS (.Time)
                          : (t.o&&g.o) ? t.o===g.o                  // hoặc cùng id lệnh aggressor (dự phòng)
                          : (!g.o && s===g.side && (t.T-g.last)<=sweepMs));  // fallback chỉ-ms
  for(const t of trades){ const s=t.side||"";
    if(same(t,s)){ g.q+=t.q; g.last=t.T; }
    else { flush(); g={T:t.T, q:t.q, side:s, tu:t.tu||0, o:t.o||"", last:t.T}; } }
  flush(); return out;
}
Tinh chỉnh: ngưỡng cố định bắn sai giữa các hợp đồng/phiên. Percentile trượt (đánh dấu size trên percentile 99 của N trade gần nhất) tự thích nghi theo thanh khoản. ATAS chồng thêm bộ lọc: vị trí giá và cỡ marker theo khối lượng đã có trong chart tham chiếu; cửa sổ giờ trong ngày và cảnh báo còn trong lộ trình. Bản phân tích parity-ATAS đầy đủ — gồm hộp sweep FirstPrice→LastPrice và vì sao AutoFilter không thể khớp 1:1 — được theo dõi trong Big trades. Tuning: a fixed threshold misfires across contracts/sessions. A rolling percentile (flag size above the 99th of the last N trades) adapts to liquidity automatically. ATAS layers more filters: price-location and marker size scaled by volume are already in the reference chart; time-of-day window and alerts are on the roadmap. The full ATAS-parity breakdown — including the FirstPrice→LastPrice sweep range and why AutoFilter can't be matched 1:1 — is tracked in Big trades.

2 · CVD — Cumulative Volume DeltaCumulative Volume Delta ✓

Trụ cột order-flow. Tổng chạy của khối lượng mua chủ động trừ bán; phân kỳ so với giá báo hiệu sự hấp thụ. The order-flow staple. Running sum of aggressive buy minus sell volume; divergence vs price signals absorption.

Dữ liệu:Data: @trade (fieldthe side) + aggTrades để seed phiênto seed the session.

Thuật toánAlgorithm

  1. cvd = 0 tại giờ mở phiên.at session open.
  2. Mỗi trade:Each trade: cvd += (side==="B" ? +q : side==="S" ? -q : signByTickRule()*q).
  3. Vẽ cvd thành đường ở sub-pane dưới giá. So độ dốc với độ dốc giá.Plot cvd as a line in a sub-pane under price. Compare slope vs price slope.
let cvd = 0, last = null; const series = [];
function onTrade(d){
  let s = d.side;
  if (s!=="B" && s!=="S"){ s = (last===null || d.p>=last) ? "B" : "S"; }  // tick rule
  last = d.p;
  cvd += (s==="B" ? d.q : -d.q);
  series.push({ time: Math.floor(d.T/1000), value: cvd });
}

3 · VWAP + dải lệch chuẩnstandard-deviation bands ✓

Đường giá trị hợp lý nội ngày được dùng nhiều nhất; dải đánh dấu trạng thái căng. The most-used intraday fair-value line; bands mark stretched conditions.

Dữ liệu:Data: @trade (price × volume) + aggTrades để seedto seed.

Thuật toánAlgorithm — giữ ba tổng chạy từ giờ mở phiên:keep three running sums from the session open:

ΣPV  += price*qty
ΣV   += qty
ΣP2V += price*price*qty
VWAP  = ΣPV / ΣV
var   = ΣP2V/ΣV − VWAP*VWAP          # volume-weighted variance
σ     = sqrt(max(var, 0))
band± = VWAP ± k*σ                    # k = 1, 2, 3 …
let pv=0,v=0,p2v=0;
function onTrade(d){ pv += d.p*d.q; v += d.q; p2v += d.p*d.p*d.q; }
function vwap(){ const m = pv/v; const sd = Math.sqrt(Math.max(p2v/v - m*m, 0));
  return { vwap:m, up1:m+sd, dn1:m-sd, up2:m+2*sd, dn2:m-2*sd }; }
Anchored VWAP = cùng công thức nhưng bắt đầu Σ từ một sự kiện chọn trước (mở phiên, một đỉnh/đáy swing, một nến tin) thay vì giờ mở phiên. Anchored VWAP = same math but start Σ from a chosen event (open, a swing high/low, a news bar) instead of the session open.

4 · Volume Profile (POC / Value Area) ✓

Histogram ngang của khối lượng theo giá — tìm node khối lượng cao (hỗ trợ/kháng cự) và vùng giá trị. Horizontal histogram of volume by price — finds high-volume nodes (support/resistance) and the value area.

Dữ liệu:Data: aggTrades (phiên/khoảng nhìn thấysession/visible range) + @trade (mở rộng liveextend live). (Hoặc tổng hợp từ footprint sẵn có.)(Or aggregate an existing footprint.)

Thuật toánAlgorithm

  1. Bucket khối lượng theo giá:Bucket volume by price: vol[price] += q cho mọi trade trong khoảng.for every trade in range.
  2. POC = giá có vol lớn nhất.price with the max vol.
  3. Vùng giá trị (VA, mặc định 70%):Value Area (VA, default 70%): bắt từ POC, lặp lại việc thêm hàng lớn hơn trong hai hàng kề (trên/dưới span VA hiện tại) đến khi khối lượng bao gồm ≥ 70% tổng. Hai mép span là VAH (cao) và VAL (thấp). start at POC, repeatedly add the larger of the two adjacent rows (above / below the current VA span) until the enclosed volume ≥ 70% of total. The span's edges are VAH (high) and VAL (low).
const vol = new Map();                                   // price -> volume
function onTrade(d){ vol.set(d.p, (vol.get(d.p)||0) + d.q); }
function valueArea(pct=0.70){
  const rows=[...vol.entries()].sort((a,b)=>a[0]-b[0]);  // ascending price
  const total=rows.reduce((s,r)=>s+r[1],0);
  let i=rows.reduce((mi,r,ix,a)=>r[1]>a[mi][1]?ix:mi,0); // POC index
  let lo=i, hi=i, acc=rows[i][1];
  while(acc < total*pct && (lo>0 || hi<rows.length-1)){
    const up   = hi<rows.length-1 ? rows[hi+1][1] : -1;
    const down = lo>0            ? rows[lo-1][1] : -1;
    if(up>=down){ hi++; acc+=up; } else { lo--; acc+=down; }
  }
  return { poc:rows[i][0], vah:rows[hi][0], val:rows[lo][0] };
}
Loại profile: theo phiên = Volume Profile; theo khoảng nhìn thấy = VPVR; tách theo bracket thời gian = TPO / Market Profile. Cùng cách bucket, khác cách nhóm trục x. Profile types: by session = Volume Profile; by visible range = VPVR; split by time brackets = TPO / Market Profile. Same bucketing, different x-grouping.

5 · DOM — thang sổ lệnhorder book ladder ✓

Thang depth mà scalper đọc: size bid/ask nghỉ tại mỗi giá, cộng mất cân bằng. The depth ladder scalpers read: resting bid/ask size at each price, plus imbalance.

Dữ liệu:Data: @depth (sổ đầy đủfull book) hoặcor @depthN (top Ntop N).

Thuật toánAlgorithm

  1. Giữ hai map bids[price]=size, asks[price]=size; thay từ mỗi snapshot @depth (hoặc áp delta nếu đăng ký incremental). Maintain two maps bids[price]=size, asks[price]=size; replace from each @depth snapshot (or apply deltas if you subscribe incremental).
  2. Vẽ thang dọc: giá giảm dần, size bid trái / size ask phải, bid/ask tốt nhất tô đậm. Render a vertical ladder: price descending, bid size left / ask size right, best bid/ask highlighted.
  3. Mất cân bằng sổBook imbalance ở đỉnh:at the top: (bidSize − askSize) / (bidSize + askSize) trên K mức đầu → áp lực.over the top K levels → pressure.
let bids=new Map(), asks=new Map();
function onDepth(d){ bids=new Map(d.bids); asks=new Map(d.asks);
  const bb=d.bids[0]?.[1]||0, aa=d.asks[0]?.[1]||0;
  const imb=(bb-aa)/((bb+aa)||1);                 // −1 sell-heavy … +1 buy-heavy
}
Size DOM là thanh khoản nghỉ (ý định), không phải khối lượng đã khớp. Ghép với @trade để thấy cái gì thực sự khớp vào sổ (hấp thụ / iceberg refill). DOM size is resting liquidity (intent), not executed volume. Pair it with @trade to see what actually traded into the book (absorption / iceberg refills).

6 · Heatmap thanh khoản (kiểu Bookmap)Liquidity Heatmap (Bookmap-style) ✓

Thanh khoản nghỉ theo thời gian — lệnh nghỉ lớn phát sáng, cho thấy nơi giá bị hút/từ chối. Resting liquidity over time — large resting orders glow, showing where price gets attracted/rejected.

Dữ liệu:Data: @depth lấy mẫu theo thời giansampled over time.

Thuật toánAlgorithm

  1. Theo nhịp cố định (vd mỗi 250 ms) chụp snapshot sổ.On a fixed cadence (e.g. every 250 ms) snapshot the book.
  2. Với mỗi [price, size], ghi một cell (timeColumn, price) = size.For each [price, size], write a cell (timeColumn, price) = size.
  3. Ánh xạ size → cường độ màu (thang log chạy tốt nhất); cuộn cột sang trái theo thời gian.Map size → color intensity (log scale works best); scroll columns left over time.

Đây là buffer 2-D (thời gian × giá) của size nghỉ. Giữ ring buffer W cột gần nhất; tô lại màu mỗi snapshot mới. Spoofing hiện ra như thanh khoản xuất hiện rồi biến mất mà không khớp xuyên qua. This is a 2-D buffer (time × price) of resting size. Keep a ring buffer of the last W columns; recolor on each new snapshot. Spoofing shows as liquidity that appears then vanishes without trading through it.

9 · Options GEX — Gamma ExposureGamma Exposure 🛠

Dữ liệu:Data: open interest quyền chọn theo từng strike (từ chuỗi) + giá underlying/future.option open interest per strike (from the chain) + the underlying/future price.

Phương pháp:Method:

  1. Với mỗi strike, tính gamma (Black-76 cho option futures):For each strike, compute gamma (Black-76 for futures options):
d1 = (ln(F/K) + 0.5σ²T) / (σ√T)
Γ  = e^(−rT)·N'(d1) / (F·σ·√T)
GEX_K = Γ · OI_K · multiplier · F² · 0.01     # notional gamma / 1% move
  1. Phơi nhiễm mỗi strike:Per-strike exposure: GEX_K = Γ · OI_K · multiplier · F² · 0.01 (gamma danh nghĩa cho 1% di chuyển).(notional gamma per 1% move).
  2. Cộng các strike với quy ước dấu dealer (thường +calls, −puts). Mức zero-gamma (điểm lật) và các strike GEX lớn đóng vai trò nam châm/ghim biến động. Sum across strikes with a dealer-sign convention (commonly +calls, −puts). The zero-gamma flip level and large-GEX strikes act as volatility magnets / pins.
Quy ước dấu (ai long/short gamma) là lựa chọn mô hình hoá — ghi rõ trong UI. The sign convention (who is long/short gamma) is a modeling choice — state it explicitly in your UI.

Bẫy chung (áp cho mọi recipe)General pitfalls (apply to all recipes)

  • Seed từ lịch sử, rồi mới live.Seed from history, then go live. Replay aggTrades một lần để điền phiên đến hiện tại, rồi tiếp tục áp @trade.
