Là gìWhat it is

Hai lớp overlay nhẹ (chủ yếu tần suất thấp) dựng trên push tham chiếu/phiên của Rithmic R|API+: overlay Market Structure (settlement, giới hạn giá ngày, trạng thái chợ, open interest, chỉ báo đấu giá) và popup Contract Spec (giá trị tick, giá trị điểm, margin, kiểu thanh toán…). Cả hai là snapshot lúc subscribe + push khi đổi → rẻ, hiện ngay, không cần replay lịch sử. Two lightweight (mostly low-frequency) overlays on Rithmic R|API+ reference/session pushes: a Market Structure overlay (settlement, daily price limits, market status, open interest, auction indicators) and a Contract Spec popup (tick value, point value, margin, settlement method…). Both are snapshot-on-subscribe + push-on-change → cheap, instant, no history replay.

Market Structure — thấy gìMarket Structure — what you see

  • Đường settlement — vàng LIỀN (Settle); dự tính vàng ĐỨT (Settle~, quanh giờ đóng).Settlement line — gold SOLID (Settle); projected gold DASHED (Settle~, near close).
  • Limit up đỏ đứt (Limit↑) / limit down xanh đứt (Limit↓). Band NaN = không vẽ.Limit up red dashed / down green dashed. A NaN band draws nothing.
  • Chỉ báo đấu giá — IOP tím (mở) / ICP xanh (đóng), chỉ trong cửa sổ đấu giá.Auction indicative — IOP (open) / ICP (close), only inside the auction window.
  • Badge cạnh nút: trạng thái chợ (Open xanh / Halt đỏ / Auction vàng / Closed xám) · OI · stl ±x (±%) (giá vs settlement, live).Badge: market status (Open green / Halt red / Auction amber / Closed grey) · OI · stl ±x (±%) (price vs settlement, live).
Settlement = giá mark chính thức của sàn (mark-to-market / mốc % ngày), KHÔNG phải last trade. OI = số hợp đồng mở, EOD (cập nhật ~1 lần/ngày, không intraday); OI cân bằng long=short theo định nghĩa, không tách bên được. Limit = band circuit-breaker, thường xa giá → phải zoom out mới thấy. Settlement = the exchange's official daily mark (mark-to-market / daily %), NOT the last trade. OI = open contracts, an EOD figure (updates ~once/day, not intraday); OI is symmetric (longs = shorts) so it can't be split by side. Limits = circuit-breaker bands, usually far from price → zoom out to see.

Bấm ⓘ cạnh mã → popup:Click ⓘ next to the symbol → popup:

  • Hợp đồng: mô tả · mã · sàn · loại · HSD.Contract: description · symbol · exchange · type · expiration.
  • Bước giá · giá trị 1 tick (= tick × point value, vd GC $10, ES $12.50) · giá trị 1 điểm ($) · tiền tệ.Tick size · tick value (= tickSize × pointValue, e.g. GC $10, ES $12.50) · point value ($) · currency.
  • Kiểu thanh toán (cash/physical) · đơn vị HĐ (× qty) · ngày GD cuối (auxRefData).Settlement method (cash/physical) · unit of measure (× qty) · last trading date (auxRefData).
  • Hạn mức RMS (productRms): giới hạn vị thế mua/bán, max lệnh, loss limit — hiện khi có (rỗng trên demo/data-only).RMS limits (productRms): buy/sell position limit, max order qty, loss limit — shown when set (empty on demo/data-only).

Thông số chính lấy từ instrumentInfo (đã cache mỗi mã) nên hiện tức thì; auxRefData/productRms điền async. Core specs come from cached instrumentInfo (instant); auxRefData/productRms fill in async.
