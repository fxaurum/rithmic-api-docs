MBO là gìWhat it is

Order-by-order depth of market dựng trên DBO (Depth-By-Order) của Rithmic. Trong khi DOM thường hiện size gộp tại mỗi giá (Level 2 / MBP), MBO hiện từng lệnh riêng tạo nên size đó — phân biệt được 1 lệnh 500 với 50 lệnh 10 ở cùng giá, thấy hàng đợi (queue/priority), và (Phase 2) bắt iceberg/spoof. An order-by-order depth of market built on Rithmic DBO (Depth-By-Order). Where the regular DOM shows the aggregated size at each price (Level 2 / MBP), MBO shows the individual orders that make up that size — telling one 500-lot order from fifty 10-lot orders at the same price, showing the queue (priority), and (Phase 2) detecting icebergs/spoofs.

Mỗi mức = [giá, số lệnh, tổng size, sizes[]]. sizes[] xếp theo queue priority (đầu hàng trước), tối đa 64 lệnh/mức. @mbo<N> cắt top N mức mỗi bên (như @depth<N>). Cửa sổ + entitlement: @mboraw đi theo CÙNG cửa sổ DBO per-price như @mbo — chỉ có event cho giá trong ±MBO_WIN_TICKS (mặc định 20) quanh BBO; sub kèm @mbo<N> để nới (cửa sổ = MAX chung). Cùng entitlement như @mbo. Tần suất CAO hơn nhiều — chỉ sub khi thực sự xử lý hết; gateway bỏ qua passthrough khi không ai giữ @mboraw. Window + entitlement: @mboraw rides the SAME per-price DBO window as @mbo — events only for prices within ±MBO_WIN_TICKS (default 20) of the BBO; subscribe @mbo<N> alongside to widen it (window = shared MAX). Same entitlement as @mbo. Much higher rate — only subscribe if you consume every event; the gateway skips the passthrough when no client holds @mboraw.

Thấy gì trên ladderWhat you see

Bật panel DOM, rồi ⚙ → "MBO — order-by-order". Cột Bid/Ask đổi từ 1 thanh đặc thành 1 block/lệnh: Turn on the DOM panel, then ⚙ → "MBO — order-by-order". The Bid/Ask columns change from one solid bar to one block per order:

  • Block từng lệnh — mỗi lệnh chờ là 1 ô, rộng ∝ size (ô cơ bản cho size-1, lớn hơn thì rộng theo tỷ lệ), vẽ theo queue từ trục giá vào (bid sang trái, ask sang phải). Số size in trong ô khi đủ rộng. Per-order blocks — each resting order is its own block, width ∝ size (a base cell for size-1, scaling up), drawn in queue order from the price axis inward (bid left, ask right). Size printed inside when wide enough.
  • ×N — số lệnh tại mức đó (vd ×4 = 4 lệnh riêng).the order count at that level (e.g. ×4 = four separate orders).
  • Vol — cột riêng cho tổng size mỗi mức (không chồng lên block nữa).a dedicated column for the per-level total (no longer overlapping the blocks).

Cách hoạt động (gateway)How it works (gateway)

DBO theo TỪNG GIÁ. subscribeDbo(exch, sym, price) sub feed order-by-order cho 1 mức giá — không có lệnh "sub cả sổ". Nên gateway duy trì cửa sổ cuộn: DBO is per-PRICE. subscribeDbo(exch, sym, price) subscribes the order-by-order feed for one price — there is no "subscribe the whole book". So the gateway maintains a rolling window:

  • MboBook/symbol: ExchOrdId → {price, size, side, priority, firstMs, lastMs}. MboBook per symbol: ExchOrdId → {price, size, side, priority, firstMs, lastMs}.
  • Timer 60 ms giữ cửa sổ sub ±20 tick quanh BBO, unsub mức trôi ra ngoài (+8 tick hysteresis chống churn). Ref-count/symbol như _mdRef — nhiều chart cùng mã dùng chung. 60 ms timer keeps the window subscribed to ±20 ticks around BBO, unsubscribing levels that drift out (+8-tick hysteresis). Ref-counted per symbol like _mdRef — many charts share.
  • OnDbo: Image/New thêm, Change sửa, Delete xoá; giá làm tròn về tick (Rithmic trả float như 4151.900000000001). OnDbo: Image/New add, Change update, Delete remove; price rounded to the tick grid (Rithmic returns floats like 4151.900000000001).
  • Mỗi nhịp gom theo giá (sort theo Priority) → emit @mbo (~20 Hz), bọc MboBox để Route cắt @mbo<N>→Top(N) (như DepthBox). Reconnect → xoá tập-giá-đã-sub để timer sub lại. Each tick aggregates by price (sorted by Priority) → emits @mbo (~20 Hz), wrapped in MboBox so Route slices @mbo<N>→Top(N) (like DepthBox). On reconnect the subscribed-price set is cleared so the timer re-subscribes.

The upgrade over the current @depth-sampled heatmap.
