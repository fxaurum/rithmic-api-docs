Tổng quanOverview

Đây là WebSocket API kiểu Binance trên một endpoint duy nhất, phát dữ liệu thị trường futures (live + lịch sử) từ Rithmic (R|API+). Client subscribe đúng stream nào thì chỉ nhận stream đó — không cần tự lọc. This is a Binance-style WebSocket API over a single endpoint that streams live and historical futures market data from Rithmic (R|API+). A client subscribes to exactly the streams it wants and receives only those — no client-side filtering needed.

Cùng cổng HTTP đó cũng phục vụ: web app chart tại GET /, tài liệu này tại GET /api.html, và hướng dẫn footprint tại GET /footprint.html. The same HTTP port also serves the chart web app at GET /, this document at GET /api.html, and the footprint guide at GET /footprint.html.

Kết nốiConnecting

Mở WebSocket tới endpoint. Không có bước handshake/login — nối xong là gửi request và subscribe được ngay. Open a WebSocket to the endpoint. There is no handshake/login step — once connected you may send requests and subscribe immediately.

ws = new WebSocket("ws://127.0.0.1:9001/");

Vòng đời kết nối & tự nối lạiConnection lifecycle & reconnect

Có 2 kết nối độc lập cần phân biệt:There are two independent connections to keep in mind:

1. WebSocket của bạn (client → gateway). Không auth. Subscription gắn với socket của bạn — socket đóng là mất. Nên nếu socket của bạn rớt, hãy nối lại và SUBSCRIBE lại (vòng retry ~2s là đủ). 1. Your WebSocket (client → gateway). No auth. Server-side subscriptions are tied to your socket — if it closes they are dropped. So if your socket drops, reconnect and re-SUBSCRIBE (a ~2 s retry loop is fine).

2. Đường upstream của gateway tới Rithmic. Gateway giữ một phiên R|API+ phục vụ mọi client. Nếu đường này rớt (chập mạng → Rithmic ConnectionBroken), gateway sẽ: 2. The gateway's upstream link to Rithmic. The gateway keeps one logged-in R|API+ session feeding all clients. If that link drops (network blip → Rithmic ConnectionBroken), the gateway:

  • tự nối lại — thử khoảng mỗi 30s tới khi được (khớp nhịp "dưới 1 phút" của Rithmic; không giới hạn số lần), và auto-reconnects — retries about every 30 s until it succeeds (matching Rithmic's sub-minute cadence; no attempt cap), and
  • khi thành công, tự phát lại MỌI subscription đang có — bạn không cần SUBSCRIBE lại. on success, auto-restores every active subscription — you do not re-SUBSCRIBE.

Trong lúc upstream gián đoạn, stream chỉ tạm dừng rồi tự chạy lại khi nối được — không có frame lỗi cho cú chập tạm thời. Request (klines, aggTrades…) gửi đúng lúc gián đoạn có thể timeout; cứ gửi lại. Gateway chỉ ngừng thử khi Rithmic từ chối đăng nhập lại (TK hết hạn/khoá) hoặc người vận hành đăng xuất. While the upstream is down, stream events simply pause and resume automatically once it reconnects — no error frame is sent for a transient blip. Requests sent during the outage may time out; just retry. The gateway stops trying only if Rithmic rejects the re-login (expired/disabled account) or the operator logs out.

Tóm lại cho client: giữ socket mở, đừng huỷ chỉ vì stream im — gateway đang tự chữa upstream và sẽ phát lại subscription. Bạn chỉ cần tự lo nối lại cho socket của mình. Takeaway: keep your socket open; don't tear down just because events went quiet — the gateway is healing the upstream and will replay your subscriptions. Only handle reconnection for your own socket.

Định dạng messageMessage format

Mỗi message bạn gửi là object JSON có method. Gắn id (số hoặc chuỗi) để khớp với response. Every message you send is a JSON object with a method. Attach a unique id (number or string) to correlate the response.

{ "method": "<name>", "params": { ... } | [ ... ], "id": 7 }

request aggTrades — trade lịch sửhistorical trades


Trả về từng lệnh khớp lịch sử kèm bên chủ động (aggressor). Đây là dữ liệu bạn replay để dựng footprint cho nến quá khứ. Ưu tiên lấy từ Rithmic replay; phiên Rithmic không replay được (chợ đóng / phiên cũ) thì rơi xuống kho tick cục bộ của gateway (trade ghi xuống đĩa lúc live) — nên footprint/big-trade còn lại qua các lần khởi động. Rỗng chỉ khi cả hai đều không có.
Returns individual historical trade prints with aggressor side. This is the data you replay to build a footprint for past candles. Served from Rithmic replay when available; for sessions Rithmic can no longer replay (market closed / older) it falls back to the gateway's local tick store (trades recorded live to disk), so footprint/big-trade history survives restarts. Empty only if neither has data.


{ "method":"aggTrades",
  "params":{ "exchange":"CME", "symbol":"ESU6",
             "startTime":1718000000000, "endTime":1718003600000, "limit":0 },
  "id": 2 }


Response — mảng [timeMs, price, qty, side, ordId, tu]; side là "B" (mua chủ động / quét ask) hoặc "S" (bán chủ động / quét bid), hoặc "" nếu không rõ. tu = mốc khớp micro giây của sàn — các print cùng matching event chia sẻ; cộng các print liên tiếp cùng tu = một cumulative trade (sweep), đúng khoá ATAS gom (CumulativeTrade.Time; xem recipes §1). ordId (AggressorExchOrdId) — Rithmic CÓ cấp nhưng gom theo nó gộp dồn lệnh khớp rải nhiều event; chỉ là dự phòng khi thiếu tu.
Response — array of [timeMs, price, qty, side, ordId, tu] where side is "B" (buy aggressor / lifted ask) or "S" (sell aggressor / hit bid), or "" if unknown. tu = exchange match time in microseconds — prints of one matching event share it; sum consecutive prints sharing tu = one cumulative trade (sweep), the key ATAS groups on (its CumulativeTrade.Time; recipes §1). ordId (AggressorExchOrdId) — Rithmic supplies it but grouping by it over-merges an order filling across several events; fallback only when tu is missing.

request bigTrades — big trade gom sẵn (server-side)cumulative big trades (server-side)


Gateway lấy tick thô trong khoảng (replay → kho local), gom theo matching event (tu → o → cửa sổ 50ms — luật ATAS), lọc min, và chỉ trả về các bóng — client vẽ big trade cả phiên mà không phải tải hàng triệu tick. Xem Big trades.
merge (ms, tùy chọn, mặc định 0): 0 = gom chính xác theo matching-event; >0 = gom kiểu ATAS cumulative — gộp MỌI lệnh cùng chiều trong cửa sổ merge ms (bất kể tu/o) → bóng to/ít hơn, giống ATAS hơn (bỏ qua cache phiên, tính tươi).
The gateway pulls raw trades for the range (replay → local store), groups by matching event (tu → o → 50 ms window — the ATAS rule), filters by min, and returns only the bubbles — a client draws a whole session's big trades without pulling millions of ticks. See Big trades.
merge (ms, optional, default 0): 0 = precise matching-event grouping; >0 = ATAS-style cumulative window — merge ALL same-side prints within merge ms regardless of tu/o → fewer, larger bubbles (bypasses the per-session cache, computed fresh).


{ "method":"bigTrades", "id":8,
  "params":{ "exchange":"CME", "symbol":"ESU6", "startTime":1718000000000, "endTime":1718003600000, "min":50, "merge":0 } }
→ { "id":8, "status":200, "result":[
  [1718000001120, "B", 271, 7612.75, 7613.00],   // [timeMs, side, vol, firstPrice, lastPrice] — vol >= min
  [1718000002360, "S", 84,  7611.50, 7611.50]    // firstPrice==lastPrice -> bóng tròn; khác nhau -> dải quét
]}

request syncTicks — tải lịch sử tick vào kho localbackfill tick history into the local store


Đưa vào hàng đợi job nền hốt tick các phiên ĐÓNG từ Rithmic (replayBars(Tick,1), phân trang) vào kho tick local, phiên gần nhất trước — để footprint/big-trade lịch sử dùng offline & sống qua restart. Trả về ngay; theo dõi tiến độ bằng syncStatus.
